# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not open a public issue.**

Instead, report it privately through GitHub's Security Advisory feature:

1. Go to the [**Security Advisories**](https://github.com/looksphishy/looksphishy/security/advisories) tab of this repository.
2. Click **"New draft security advisory"**.
3. Fill in the details of the vulnerability, including steps to reproduce.
4. Submit the advisory — only you and the repository maintainers will be able to see it.

You can also use the **"Report a vulnerability"** button on the [Security](https://github.com/looksphishy/looksphishy/security) tab if private vulnerability reporting is enabled.

## What to Expect

- We will acknowledge your report within **3 business days**.
- We will work with you to understand and validate the issue.
- We aim to release a fix within **30 days** of confirmation, depending on severity and complexity.
- We will credit you in the advisory (unless you prefer to remain anonymous).

## Scope

This policy covers the code in this repository. If you find a vulnerability on the live looksphishy.org service that isn't related to this codebase, please still report it via the process above.

## Thank You

We appreciate the security research community's efforts in helping keep LooksPhishy and its users safe.
